# about_DRMMAlert

## SHORT DESCRIPTION

Describes the DRMMAlert class and its methods for managing alerts in Datto RMM.

## LONG DESCRIPTION

The DRMMAlert class represents an alert generated by Datto RMM monitoring. Alerts are notifications of issues, warnings, or events detected on managed devices or infrastructure.

DRMMAlert objects are returned by Get-RMMAlert and provide a comprehensive object-oriented interface for alert management and analysis.

## PROPERTIES

The DRMMAlert class exposes the following properties:

| Property | Type | Description |
|----------|------|-------------|
| AlertUid | [guid] | Alert unique identifier |
| Priority | [string] | Alert priority: 'Critical', 'High', 'Moderate', 'Low', 'Information' |
| Diagnostics | [string] | Diagnostic information and details |
| Resolved | [bool] | Whether alert has been resolved |
| ResolvedBy | [string] | User who resolved the alert |
| ResolvedOn | [Nullable[datetime]] | When alert was resolved |
| Muted | [bool] | Whether alert is muted |
| TicketNumber | [string] | Associated ticket number |
| Timestamp | [Nullable[datetime]] | When alert was created |
| AlertMonitorInfo | [DRMMAlertMonitorInfo] | Information about the monitor that triggered the alert |
| AlertContext | [DRMMAlertContext] | Type-specific context data (varies by alert type) |
| AlertSourceInfo | [DRMMAlertSourceInfo] | Information about the alert source (device/site) |
| ResponseActions | [DRMMAlertResponseAction[]] | Automated response actions triggered |
| AutoresolveMins | [Nullable[int]] | Auto-resolution timeout in minutes |
| SiteUid | [Nullable[guid]] | Site GUID (when queried at site scope) |

## METHODS

### Status Checking

#### IsOpen()

Returns whether the alert is currently open (not resolved).

**Returns:** [bool]

**Example:**
```powershell
$alert = Get-RMMAlert | Select-Object -First 1
if ($alert.IsOpen()) {
    Write-Host "Alert $($alert.AlertUid) is still open"
}
```

#### IsCritical()

Returns whether the alert has Critical priority.

**Returns:** [bool]

**Example:**
```powershell
$alerts = Get-RMMAlert
$criticalAlerts = $alerts | Where-Object {$_.IsCritical()}
```

#### IsHigh()

Returns whether the alert has High priority.

**Returns:** [bool]

**Example:**
```powershell
$alerts = Get-RMMAlert
$highPriorityAlerts = $alerts | Where-Object {$_.IsHigh()}
```

### Alert Management

#### Resolve()

Resolves (closes) the alert.

**Returns:** [void]

**Example:**
```powershell
$alert = Get-RMMAlert -DeviceUid $deviceUid | Where-Object {$_.IsOpen()}
foreach ($a in $alert) {
    $a.Resolve()
}
```

### Utility Methods

#### GetSummary()

Returns a formatted summary string of the alert including status, priority, device name, and monitor information.

**Returns:** [string]

**Example:**
```powershell
$alerts = Get-RMMAlert
foreach ($alert in $alerts) {
    Write-Host $alert.GetSummary()
}
# Output examples:
# [Open] Critical - SERVER01 - Antivirus: Definitions out of date
# [Resolved] High - WORKSTATION05 - Disk: Low disk space on C:
# [Open (Muted)] Moderate - LAPTOP10 - EventLog: Application error detected
```

## RELATED CLASSES

### DRMMAlertMonitorInfo

Contains information about the monitoring component that triggered the alert.

**Properties:**
- SendsEmails [bool] - Whether monitor sends email notifications
- CreatesTicket [bool] - Whether monitor creates tickets

### DRMMAlertContext

Base class for alert context information. The actual type varies based on alert category.

**Derived Types:**
- DRMMAlertContextAntivirus - Antivirus-related alerts
- DRMMAlertContextBackupManagement - Backup alerts
- DRMMAlertContextDiskHealth - Disk health alerts
- DRMMAlertContextDiskUsage - Disk space alerts
- DRMMAlertContextEventLog - Windows Event Log alerts
- DRMMAlertContextGeneric - Generic/untyped alerts
- DRMMAlertContextOnlineOfflineStatus - Device online/offline alerts
- DRMMAlertContextPatch - Windows Update/patching alerts
- DRMMAlertContextResourceUsage - CPU/Memory usage alerts
- DRMMAlertContextScript - Script/component execution alerts
- And many more...

### DRMMAlertSourceInfo

Contains information about the source of the alert.

**Properties:**
- DeviceUid [string] - Device unique identifier
- DeviceName [string] - Device hostname
- SiteUid [string] - Site unique identifier
- SiteName [string] - Site name

### DRMMAlertResponseAction

Represents an automated action taken in response to the alert.

**Properties:**
- ActionTime [Nullable[datetime]] - When action was executed
- ActionType [string] - Type of response action
- Description [string] - Action description
- ActionReference [string] - Reference identifier
- ActionReferenceInt [string] - Internal reference

## EXAMPLES

### Example 1: Get all open critical alerts

```powershell
$criticalAlerts = Get-RMMAlert | Where-Object {$_.IsOpen() -and $_.IsCritical()}
Write-Host "Found $($criticalAlerts.Count) open critical alerts"
foreach ($alert in $criticalAlerts) {
    Write-Host $alert.GetSummary()
}
```

### Example 2: Resolve all alerts for a specific device

```powershell
$device = Get-RMMDevice -Hostname "SERVER01"
$openAlerts = Get-RMMAlert -DeviceUid $device.Uid | Where-Object {$_.IsOpen()}
foreach ($alert in $openAlerts) {
    $alert.Resolve()
    Write-Host "Resolved: $($alert.GetSummary())"
}
```

### Example 3: Group alerts by priority

```powershell
$alerts = Get-RMMAlert
$grouped = $alerts | Group-Object Priority | Sort-Object Name
foreach ($group in $grouped) {
    Write-Host "$($group.Name): $($group.Count) alerts"
}
```

### Example 4: Find muted alerts

```powershell
$mutedAlerts = Get-RMMAlert | Where-Object {$_.Muted}
Write-Host "Muted alerts:"
$mutedAlerts | ForEach-Object {
    Write-Host "  $($_.AlertSourceInfo.DeviceName) - $($_.Priority)"
}
```

### Example 5: Export alert details to CSV

```powershell
$alerts = Get-RMMAlert
$alerts | Select-Object `
    AlertUid,
    Priority,
    @{N='DeviceName';E={$_.AlertSourceInfo.DeviceName}},
    @{N='SiteName';E={$_.AlertSourceInfo.SiteName}},
    Timestamp,
    Resolved,
    Muted | Export-Csv -Path "alerts.csv" -NoTypeInformation
```
