TOPIC
    about_DRMMAlert

SHORT DESCRIPTION
    Describes the DRMMAlert class and its methods for managing alerts in Datto RMM.

LONG DESCRIPTION
    The DRMMAlert class represents an alert generated by Datto RMM monitoring. Alerts
    are notifications of issues, warnings, or events detected on managed devices or
    infrastructure.
    
    DRMMAlert objects are returned by Get-RMMAlert and provide a comprehensive
    object-oriented interface for alert management and analysis.

PROPERTIES
    The DRMMAlert class exposes the following properties:

    AlertUid            [guid]                      - Alert unique identifier
    Priority            [string]                    - Alert priority: 'Critical', 'High', 'Moderate', 'Low', 'Information'
    Diagnostics         [string]                    - Diagnostic information and details
    Resolved            [bool]                      - Whether alert has been resolved
    ResolvedBy          [string]                    - User who resolved the alert
    ResolvedOn          [Nullable[datetime]]        - When alert was resolved
    Muted               [bool]                      - Whether alert is muted
    TicketNumber        [string]                    - Associated ticket number
    Timestamp           [Nullable[datetime]]        - When alert was created
    AlertMonitorInfo    [DRMMAlertMonitorInfo]      - Information about the monitor that triggered the alert
    AlertContext        [DRMMAlertContext]          - Type-specific context data (varies by alert type)
    AlertSourceInfo     [DRMMAlertSourceInfo]       - Information about the alert source (device/site)
    ResponseActions     [DRMMResponseAction[]]      - Automated response actions triggered
    AutoresolveMins     [Nullable[int]]             - Auto-resolution timeout in minutes
    SiteUid             [Nullable[guid]]            - Site GUID (when queried at site scope)

METHODS
    Status Checking
    ---------------
    IsOpen()
        Returns whether the alert is currently open (not resolved).
        
        Returns: [bool]
        
        Example:
            $alert = Get-RMMAlert | Select-Object -First 1
            if ($alert.IsOpen()) {
                Write-Host "Alert $($alert.AlertUid) is still open"
            }

    IsCritical()
        Returns whether the alert has Critical priority.
        
        Returns: [bool]
        
        Example:
            $alerts = Get-RMMAlert
            $criticalAlerts = $alerts | Where-Object {$_.IsCritical()}

    IsHigh()
        Returns whether the alert has High priority.
        
        Returns: [bool]
        
        Example:
            $alerts = Get-RMMAlert
            $highPriorityAlerts = $alerts | Where-Object {$_.IsHigh()}

    Alert Management
    ----------------
    Resolve()
        Resolves (closes) the alert.
        
        Returns: [void]
        
        Example:
            $alert = Get-RMMAlert -DeviceUid $deviceUid | Where-Object {$_.IsOpen()}
            foreach ($a in $alert) {
                $a.Resolve()
            }

    Utility Methods
    ---------------
    GetSummary()
        Returns a formatted summary string of the alert including status, priority,
        device name, and monitor information.
        
        Returns: [string]
        
        Example:
            $alerts = Get-RMMAlert
            foreach ($alert in $alerts) {
                Write-Host $alert.GetSummary()
            }
            # Output examples:
            # [Open] Critical - SERVER01 - Antivirus: Definitions out of date
            # [Resolved] High - WORKSTATION05 - Disk: Low disk space on C:
            # [Open (Muted)] Moderate - LAPTOP10 - EventLog: Application error detected

RELATED CLASSES
    DRMMAlertMonitorInfo
        Contains information about the monitoring component that triggered the alert.
        
        Properties:
            - Category      [string] - Monitor category (e.g., "Antivirus", "Disk", "EventLog")
            - Description   [string] - Monitor description
            - MonitorUid    [guid]   - Monitor unique identifier
            - SnoozeUntil   [string] - Snooze expiration time

    DRMMAlertContext
        Base class for alert-specific context information. Actual type varies by alert
        category and provides specialized properties for each alert type.
        
        Derived types include:
            - DRMMAlertContextAntivirus
            - DRMMAlertContextDiskUsage
            - DRMMAlertContextEventLog
            - DRMMAlertContextPatch
            - DRMMAlertContextResourceUsage
            - And many others (see notes)

    DRMMAlertSourceInfo
        Contains information about where the alert originated.
        
        Properties:
            - DeviceUid     [guid]   - Source device UID
            - DeviceId      [long]   - Source device ID
            - DeviceName    [string] - Source device name
            - SiteUid       [guid]   - Source site UID
            - SiteName      [string] - Source site name

    DRMMResponseAction
        Represents an automated response action triggered by the alert.
        
        Properties:
            - Name          [string] - Action name
            - Description   [string] - Action description
            - Status        [string] - Execution status
            - JobUid        [guid]   - Associated job UID (if applicable)

METHOD CHAINING
    DRMMAlert methods can be used in PowerShell pipelines and combined with other
    classes for comprehensive alert management workflows:

    Example: Resolve all critical alerts on offline devices
        $offlineDevices = Get-RMMDevice | Where-Object {-not $_.Online}
        foreach ($device in $offlineDevices) {
            $alerts = $device.GetAlerts() | Where-Object {$_.IsCritical() -and $_.IsOpen()}
            foreach ($alert in $alerts) {
                Write-Host "Resolving: $($alert.GetSummary())"
                $alert.Resolve()
            }
        }

    Example: Filter and resolve alerts by site
        $site = Get-RMMSite -Name "Remote Office"
        $alerts = $site.GetAlerts('Open') | Where-Object {$_.Priority -eq 'Moderate'}
        $alerts | ForEach-Object {$_.Resolve()}

EXAMPLES
    Example 1: Get summary of all open critical alerts
        $alerts = Get-RMMAlert | Where-Object {$_.IsCritical() -and $_.IsOpen()}
        foreach ($alert in $alerts) {
            Write-Host $alert.GetSummary()
            Write-Host "  Device: $($alert.AlertSourceInfo.DeviceName)"
            Write-Host "  Site: $($alert.AlertSourceInfo.SiteName)"
            Write-Host "  Monitor: $($alert.AlertMonitorInfo.Category) - $($alert.AlertMonitorInfo.Description)"
            Write-Host "  Time: $($alert.Timestamp)"
            Write-Host ""
        }

    Example 2: Resolve all muted alerts
        $mutedAlerts = Get-RMMAlert | Where-Object {$_.Muted -and $_.IsOpen()}
        Write-Host "Found $($mutedAlerts.Count) muted open alerts"
        
        foreach ($alert in $mutedAlerts) {
            Write-Host "Resolving muted alert: $($alert.GetSummary())"
            $alert.Resolve()
        }

    Example 3: Group alerts by priority
        $alerts = Get-RMMAlert
        $grouped = $alerts | Group-Object -Property Priority
        
        foreach ($group in $grouped | Sort-Object Count -Descending) {
            Write-Host "$($group.Name): $($group.Count) alerts"
            $open = ($group.Group | Where-Object {$_.IsOpen()}).Count
            Write-Host "  Open: $open"
        }

    Example 4: Find alerts with response actions
        $alerts = Get-RMMAlert | Where-Object {$_.ResponseActions.Count -gt 0}
        
        foreach ($alert in $alerts) {
            Write-Host $alert.GetSummary()
            foreach ($action in $alert.ResponseActions) {
                Write-Host "  Action: $($action.Name) - Status: $($action.Status)"
            }
        }

    Example 5: Export open alerts to CSV
        Get-RMMAlert | Where-Object {$_.IsOpen()} | Select-Object `
            @{N='Device';E={$_.AlertSourceInfo.DeviceName}},
            @{N='Site';E={$_.AlertSourceInfo.SiteName}},
            Priority,
            @{N='Monitor';E={$_.AlertMonitorInfo.Description}},
            Timestamp,
            Muted |
        Export-Csv -Path "open_alerts.csv" -NoTypeInformation

    Example 6: Resolve old information alerts
        $cutoffDate = (Get-Date).AddDays(-7)
        $oldInfoAlerts = Get-RMMAlert | Where-Object {
            $_.Priority -eq 'Information' -and
            $_.IsOpen() -and
            $_.Timestamp -lt $cutoffDate
        }
        
        Write-Host "Resolving $($oldInfoAlerts.Count) old information alerts"
        $oldInfoAlerts | ForEach-Object {$_.Resolve()}

    Example 7: Alert analysis by device
        $site = Get-RMMSite -Name "Main Office"
        $devices = $site.GetDevices()
        
        foreach ($device in $devices) {
            $alerts = $device.GetAlerts('Open')
            if ($alerts.Count -gt 0) {
                Write-Host "$($device.Hostname): $($alerts.Count) open alerts"
                $critical = ($alerts | Where-Object {$_.IsCritical()}).Count
                if ($critical -gt 0) {
                    Write-Host "  ** $critical CRITICAL **" -ForegroundColor Red
                }
            }
        }

BEST PRACTICES
    1. Use priority checking methods (IsCritical(), IsHigh()) rather than string
       comparison for cleaner, more maintainable code.

    2. Always check IsOpen() before calling Resolve() to avoid unnecessary API calls
       on already-resolved alerts.

    3. Use GetSummary() for logging and reporting rather than constructing alert
       descriptions manually.

    4. When bulk-resolving alerts, consider filtering by age and priority to avoid
       accidentally resolving recent critical issues.

    5. Leverage AlertContext for alert-specific details. Cast to the appropriate
       context type based on AlertMonitorInfo.Category for full access to
       specialized properties.

    6. Check for ResponseActions to understand what automated remediation has been
       attempted before manual intervention.

    7. Use the Muted property to identify alerts that have been intentionally
       suppressed but may still need attention.

    8. Cache alert collections when performing multiple operations to reduce API calls:
           $alerts = Get-RMMAlert
           $openCount = ($alerts | Where-Object {$_.IsOpen()}).Count
           $criticalCount = ($alerts | Where-Object {$_.IsCritical()}).Count

NOTES
    - Alert priority levels: Information, Low, Moderate, High, Critical

    - AlertContext is a base class. The actual type returned depends on the alert
      category and includes specialized subclasses like:
      * DRMMAlertContextAntivirus (antivirus alerts)
      * DRMMAlertContextDiskUsage (disk space alerts)
      * DRMMAlertContextEventLog (Windows Event Log alerts)
      * DRMMAlertContextPatch (patch management alerts)
      * DRMMAlertContextResourceUsage (CPU/Memory alerts)
      * DRMMAlertContextOnlineOfflineStatus (device connectivity)
      * And many more specialized context types

    - The Resolve() method calls Resolve-RMMAlert internally using the alert's UID.

    - Some alerts may auto-resolve based on AutoresolveMins if the condition clears.

    - The SiteUid property is only populated when alerts are retrieved at site scope
      (e.g., via Get-RMMSite | Get-RMMAlert).
