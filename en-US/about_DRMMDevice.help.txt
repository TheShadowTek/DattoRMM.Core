TOPIC
    about_DRMMDevice

SHORT DESCRIPTION
    Describes the DRMMDevice class and its methods for managing devices in Datto RMM.

LONG DESCRIPTION
    The DRMMDevice class represents a managed device within Datto RMM. Devices are
    endpoints (servers, workstations, network devices, etc.) monitored and managed
    through the Datto RMM platform.
    
    DRMMDevice objects are returned by Get-RMMDevice and provide a comprehensive
    object-oriented interface for device management, monitoring, and automation.

PROPERTIES
    The DRMMDevice class exposes the following properties:

    Id                          [long]                      - Numeric device identifier
    Uid                         [guid]                      - Device GUID
    SiteId                      [long]                      - Site numeric ID
    SiteUid                     [guid]                      - Site GUID
    SiteName                    [string]                    - Site name
    DeviceType                  [DRMMDeviceType]            - Device type information
    Hostname                    [string]                    - Device hostname
    IntIpAddress                [string]                    - Internal IP address
    OperatingSystem             [string]                    - Operating system
    LastLoggedInUser            [string]                    - Last logged in user (masked by default)
    Domain                      [string]                    - Domain name
    CagVersion                  [string]                    - Agent version
    DisplayVersion              [string]                    - Display version
    ExtIpAddress                [string]                    - External IP address
    Description                 [string]                    - Device description
    A64Bit                      [bool]                      - Whether 64-bit
    RebootRequired              [bool]                      - Reboot required flag
    Online                      [bool]                      - Online status
    Suspended                   [bool]                      - Suspended status
    Deleted                     [bool]                      - Deleted flag
    LastSeen                    [Nullable[datetime]]        - Last seen date
    LastReboot                  [Nullable[datetime]]        - Last reboot date
    LastAuditDate               [Nullable[datetime]]        - Last audit date
    CreationDate                [Nullable[datetime]]        - Creation date
    Udfs                        [DRMMDeviceUdfs]            - User-defined fields
    SnmpEnabled                 [bool]                      - SNMP enabled flag
    DeviceClass                 [string]                    - Device class
    PortalUrl                   [string]                    - Portal URL
    WarrantyDate                [string]                    - Warranty expiry date
    Antivirus                   [DRMMDeviceAntivirusInfo]   - Antivirus information
    PatchManagement             [DRMMDevicePatchManagement] - Patch management status
    SoftwareStatus              [string]                    - Software status
    WebRemoteUrl                [string]                    - Web remote URL
    NetworkProbe                [bool]                      - Network probe flag
    OnboardedViaNetworkMonitor  [bool]                      - Onboarded via network monitor
    RevealLastLoggedInUser      [bool]                      - Whether last user is revealed

METHODS
    Alert Management
    ----------------
    GetAlerts()
        Retrieves all alerts for the device.
        
        Returns: [DRMMAlert[]]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $alerts = $device.GetAlerts()

    GetAlerts([string]$Status)
        Retrieves alerts with specific status for the device.
        
        Parameters:
            $Status - Alert status: 'Open', 'Resolved', or 'All'
        
        Returns: [DRMMAlert[]]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $openAlerts = $device.GetAlerts('Open')

    ResolveAllAlerts()
        Resolves all open alerts for the device.
        
        Returns: [void]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $device.ResolveAllAlerts()
            Write-Host "All alerts resolved for $($device.Hostname)"

    Data Retrieval
    --------------
    GetAudit()
        Retrieves comprehensive audit data for the device including hardware,
        software, and system information.
        
        Returns: [DRMMDeviceAudit]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $audit = $device.GetAudit()
            Write-Host "RAM: $($audit.SystemInfo.TotalPhysicalMemory / 1GB) GB"

    GetSoftware()
        Retrieves installed software inventory for the device.
        
        Returns: [DRMMDeviceAuditSoftware[]]
        
        Example:
            $device = Get-RMMDevice -Hostname "WORKSTATION05"
            $software = $device.GetSoftware()
            $office = $software | Where-Object {$_.Name -like "*Microsoft Office*"}

    UDF Management
    --------------
    GetUdfAsJson([int]$UdfNumber)
        Parses a UDF as JSON and returns the object.
        
        Parameters:
            $UdfNumber - UDF number (1-30)
        
        Returns: [object]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $config = $device.GetUdfAsJson(5)
            Write-Host "Backup server: $($config.BackupServer)"

    GetUdfAsCsv([int]$UdfNumber, [string]$Delimiter, [string[]]$ColumnHeaders)
        Parses a delimited UDF as CSV and returns a PSCustomObject.
        
        Parameters:
            $UdfNumber     - UDF number (1-30)
            $Delimiter     - Delimiter character
            $ColumnHeaders - Array of column names
        
        Returns: [pscustomobject]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $data = $device.GetUdfAsCsv(10, '|', @('Name', 'Value', 'Status'))

    SetUDF([hashtable]$UDFFields)
        Updates one or more UDF fields for the device.
        
        Parameters:
            $UDFFields - Hashtable of UDF names and values (e.g., @{UDF1='Value'})
        
        Returns: [DRMMDevice]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $device.SetUDF(@{
                UDF1 = 'Production'
                UDF2 = 'East Datacenter'
                UDF3 = 'Critical'
            })

    ClearUDF([int]$UdfNumber)
        Clears a specific UDF field.
        
        Parameters:
            $UdfNumber - UDF number to clear (1-30)
        
        Returns: [DRMMDevice]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $device.ClearUDF(5)

    ClearUDFs()
        Clears all 30 UDF fields for the device.
        
        Returns: [DRMMDevice]
        
        Example:
            $device = Get-RMMDevice -Hostname "TEMP-DEVICE"
            $device.ClearUDFs()

    Device Management
    -----------------
    SetWarranty([datetime]$WarrantyDate)
        Updates the warranty expiry date for the device.
        
        Parameters:
            $WarrantyDate - Warranty expiry date
        
        Returns: [DRMMDevice]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $device.SetWarranty((Get-Date).AddYears(3))

    Move([guid]$TargetSiteUid)
        Moves the device to a different site.
        
        Parameters:
            $TargetSiteUid - GUID of the target site
        
        Returns: [DRMMDevice]
        
        Example:
            $device = Get-RMMDevice -Hostname "LAPTOP10"
            $newSite = Get-RMMSite -Name "Remote Office"
            $device.Move($newSite.Uid)

    Job Execution
    -------------
    RunQuickJob([guid]$ComponentUid, [hashtable]$Variables)
        Executes a quick job on the device with specified variables.
        
        Parameters:
            $ComponentUid - GUID of the component/script to run
            $Variables    - Hashtable of job variables
        
        Returns: [DRMMJob]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $compUid = (Get-RMMComponent -Name "Restart Service").Uid
            $job = $device.RunQuickJob($compUid, @{ServiceName='W32Time'})

    Utility Methods
    ---------------
    OpenPortal()
        Opens the device's portal page in the default browser.
        
        Returns: [void]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            $device.OpenPortal()

    OpenWebRemote()
        Opens the device's web remote session in the default browser.
        
        Returns: [void]
        
        Example:
            $device = Get-RMMDevice -Hostname "WORKSTATION05"
            $device.OpenWebRemote()

    GetSummary()
        Returns a formatted summary string of the device.
        
        Returns: [string]
        
        Example:
            $device = Get-RMMDevice -Hostname "SERVER01"
            Write-Host $device.GetSummary()
            # Output: SERVER01|Server

RELATED CLASSES
    DRMMDeviceType
        Contains device type and category information.
        
        Properties:
            - Category  [string] - Device category (e.g., "Server", "Workstation")
            - Type      [string] - Specific device type

    DRMMDeviceUdfs
        Contains all 30 user-defined fields for the device.
        
        Properties:
            - Udf1 through Udf30  [string] - User-defined field values

    DRMMDeviceAntivirusInfo
        Contains antivirus status information.
        
        Properties:
            - AntivirusProduct     [string] - Antivirus product name
            - DefinitionDate       [string] - Definition date
            - Status               [string] - Antivirus status

    DRMMDevicePatchManagement
        Contains Windows Update and patch management status.
        
        Properties:
            - PatchStatus          [string] - Overall patch status
            - PolicyName           [string] - Applied policy name

    DRMMDeviceAudit
        Comprehensive audit data including hardware and software inventory.
        Returned by GetAudit() method.

    DRMMDeviceAuditSoftware
        Individual software installation record.
        Returned by GetSoftware() method.

METHOD CHAINING
    DRMMDevice methods return typed objects that support method chaining and
    integration with other classes:

    Example: Get device, resolve alerts, update UDF
        $device = Get-RMMDevice -Hostname "SERVER01"
        $device.ResolveAllAlerts()
        $device.SetUDF(@{UDF1 = 'Maintenance Complete'})

    Example: Filter devices and execute job
        $site = Get-RMMSite -Name "Production"
        $servers = $site.GetFilter("Windows Servers").GetDevices()
        $compUid = (Get-RMMComponent -Name "Windows Update").Uid
        foreach ($server in $servers) {
            $server.RunQuickJob($compUid, @{})
        }

    Example: Bulk UDF clearing
        Get-RMMDevice -SiteName "Test Lab" | ForEach-Object {
            $_.ClearUDFs()
        }

EXAMPLES
    Example 1: Resolve all alerts for offline devices
        $offlineDevices = Get-RMMDevice | Where-Object {-not $_.Online}
        foreach ($device in $offlineDevices) {
            $alertCount = ($device.GetAlerts('Open')).Count
            if ($alertCount -gt 0) {
                Write-Host "$($device.Hostname): Resolving $alertCount alerts"
                $device.ResolveAllAlerts()
            }
        }

    Example 2: Update warranty dates for all servers
        $servers = Get-RMMDevice -DeviceType "Server"
        $warrantyDate = (Get-Date).AddYears(3)
        foreach ($server in $servers) {
            $server.SetWarranty($warrantyDate)
        }

    Example 3: Parse JSON configuration from UDF
        $device = Get-RMMDevice -Hostname "APP-SERVER-01"
        $config = $device.GetUdfAsJson(5)
        
        Write-Host "Environment: $($config.Environment)"
        Write-Host "Backup Schedule: $($config.BackupSchedule)"

    Example 4: Tag devices by operating system
        $devices = Get-RMMDevice
        foreach ($device in $devices) {
            $osTag = switch -Wildcard ($device.OperatingSystem) {
                '*Windows Server*' { 'Server' }
                '*Windows 10*'     { 'Win10' }
                '*Windows 11*'     { 'Win11' }
                default            { 'Other' }
            }
            $device.SetUDF(@{UDF1 = $osTag})
        }

    Example 5: Get audit data and check disk space
        $device = Get-RMMDevice -Hostname "FILE-SERVER-01"
        $audit = $device.GetAudit()
        
        foreach ($disk in $audit.LogicalDisks) {
            $freePercent = ($disk.FreeSpace / $disk.Size) * 100
            Write-Host "$($disk.DeviceId): $([math]::Round($freePercent, 2))% free"
        }

    Example 6: Execute patching job on filtered devices
        $site = Get-RMMSite -Name "Production"
        $servers = $site.GetFilter("Windows Servers").GetDevices()
        $patchComponent = Get-RMMComponent -Name "Install Windows Updates"
        
        foreach ($server in $servers) {
            Write-Host "Patching $($server.Hostname)"
            $job = $server.RunQuickJob($patchComponent.Uid, @{})
            Write-Host "  Job ID: $($job.Uid)"
        }

    Example 7: Move devices between sites
        $sourceSite = Get-RMMSite -Name "Old Office"
        $targetSite = Get-RMMSite -Name "New Office"
        $devices = Get-RMMDevice -SiteUid $sourceSite.Uid
        
        foreach ($device in $devices) {
            Write-Host "Moving $($device.Hostname) to $($targetSite.Name)"
            $device.Move($targetSite.Uid)
        }

BEST PRACTICES
    1. Use ResolveAllAlerts() for bulk maintenance operations, but review alerts
       individually for production systems to avoid masking real issues.

    2. Always validate UDF content before parsing with GetUdfAsJson() or
       GetUdfAsCsv(). Use try/catch blocks to handle parsing errors gracefully.

    3. When using SetUDF(), only include the UDFs you want to change. The method
       preserves existing UDF values not specified in the hashtable.

    4. Use ClearUDF() for individual fields during cleanup, but use ClearUDFs()
       sparingly as it wipes all 30 fields at once.

    5. Cache device objects when performing multiple operations to avoid repeated
       API calls:
           $device = Get-RMMDevice -Hostname "SERVER01"
           $alerts = $device.GetAlerts('Open')
           $audit = $device.GetAudit()
           $software = $device.GetSoftware()

    6. When moving devices with Move(), ensure the target site has appropriate
       policies and settings configured before the migration.

    7. For RunQuickJob(), use Get-RMMComponent to resolve component names to UIDs
       rather than hardcoding GUIDs.

    8. The LastLoggedInUser property is masked by default for privacy. Use the
       -IncludeLastLoggedInUser parameter with Get-RMMDevice if you need the
       actual username.

NOTES
    - All write operations (SetUDF, ClearUDF, ClearUDFs, SetWarranty, Move,
      RunQuickJob, ResolveAllAlerts) use the -Force parameter internally to
      bypass confirmation prompts, making them suitable for automation.

    - Methods that call module functions include error checking to ensure the
      required commands are available.

    - The DRMMDevice class inherits from DRMMObject, providing common functionality
      across all Datto RMM object types.

    - GetAudit() and GetSoftware() methods make separate API calls and may take
      longer than simple property access. Cache results when possible.
